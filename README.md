# Tech Scout Radar Challenge

A single-page browser game for a classroom activity on **Technology Intelligence and Scouting**. Teams act as Technology Managers, evaluate technology signals, and place each signal into a technology radar ring:

- Adopt
- Trial
- Assess
- Hold

The activity helps students practice technology assessment, technology scouting, strategic fit analysis, maturity evaluation, risk analysis, and business impact reasoning.

## Features

- Single-file HTML, CSS, and JavaScript game
- No installation or build step required
- Team-based gameplay using one device per group
- Quick Sprint mode with 6 randomized signals
- Full Challenge mode with 12 randomized signals
- Configurable 10, 12, or 15 minute timer
- Skip-and-return flow while the timer is running
- Scoring for correct answers, near answers, rationale quality, and speed
- Final score screen designed for screenshot submission
- Correct answer review after finishing the challenge

## How To Run

Open this file directly in a browser:

```bash
tech_scouting_radar_game.html
```

No local server is required.

## How To Play

1. Enter a team name.
2. Choose a game mode and timer length.
3. Read each technology signal and supporting evidence.
4. Choose the best radar ring: Adopt, Trial, Assess, or Hold.
5. Write a short rationale using concepts such as impact, risk, maturity, customer value, cost, privacy, security, governance, or strategic fit.
6. Save the decision and move to the next signal.
7. Finish once all signals are answered.
8. Screenshot the final score and answer review screen.

## Scoring

- Correct radar ring: +10
- Near answer: +5
- Rationale bonus: up to +3
- Speed bonus: up to +10 when all signals are answered

After clicking **Finish & Show Score**, the result modal displays the team score, correct decision count, time used, and the correct answer for each signal.

## Facilitator Notes

The final screen includes an answer key so the activity can move directly into discussion. Ask each team to explain:

- One technology decision they were confident about
- One technology signal they skipped, changed, or found difficult
- How they balanced value, maturity, strategic fit, and risk
